Setup

Installation

LindbladTTN is a pure-Python package built on top of PyTorch. The same install works on CPU and on CUDA-enabled GPUs.

Requirements

  • Python ≥ 3.11
  • PyTorch ≥ 2.0 — tensor operations, autograd, GPU support
  • torchdiffeq ≥ 0.2 — adaptive ODE integration (used by the VMF strategy)
  • numpy / scipy — operator construction, input handling
  • tqdm — progress bars during propagation

Optional for the examples and notebooks:

  • matplotlib ≥ 3.7 — plotting
  • qutip ≥ 5.0 — side-by-side validation against the standard open-system solver

Install from source

Clone the repository and install in editable mode:

git clone https://github.com/<your-org>/lindblad_ttn.git
cd lindblad_ttn
pip install -e .

The editable install (-e .) lets you modify the source and have the changes picked up immediately without reinstalling.

Why source-only?

LindbladTTN is still pre-1.0. A PyPI release will follow once the API stabilises and the accompanying paper is out.

GPU setup

If you have a CUDA-capable GPU, install the matching PyTorch wheel before installing LindbladTTN. Pick the right command from pytorch.org for your CUDA version. For example, CUDA 12.1:

pip install torch --index-url https://download.pytorch.org/whl/cu121
pip install -e .

Then pass device="cuda" when constructing the solver:

solver = LindbladTTN(
    H0=H0, L_ops=L_ops, n_sites=8, bond_dim=64,
    device="cuda", dtype="complex64",   # 3x faster on GPU than complex128
)

complex64 trades ~10⁻⁵ accuracy for ~3× speedup. Stay with complex128 on CPU.

Running the examples

Install the optional plotting and validation packages, then run any script in examples/:

pip install matplotlib qutip
python examples/qutip_simple.py

Each example prints a max-error metric against qutip.mesolve and saves a PNG comparison plot next to the script.

Running the tests

pip install pytest
pytest tests/

The test suite validates LindbladTTN's outputs against QuTiP for the single-qubit, driven, and small multi-qubit cases.
