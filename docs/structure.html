Architecture

Code Structure

LindbladTTN is organised in three independent layers under a single user-facing class.

Project layout

lindblad_ttn/
├── __init__.py          # Public exports: LindbladTTN, LindbladResult
├── solver.py            # Main entry point — the only class users touch
├── time_dependent.py    # H(t) = H0 + Σᵢ fᵢ(t)·Vᵢ composition
├── core/                # ← Layer 1: TTN infrastructure
│   ├── backend.py
│   ├── graph.py
│   ├── model.py
│   ├── sop.py
│   └── gauge.py
├── physics/             # ← Layer 2: Lindblad master equation
│   ├── liouvillian.py
│   ├── liouville.py
│   └── frame_factory.py
└── propagation/         # ← Layer 3: TDVP time evolution
    ├── tdvp.py
    ├── effective_h.py
    └── integrators.py

Layer 1 — core/

The TTN infrastructure. Everything in core/ is physics-agnostic — it knows about tensors, bonds, and tree topologies, not about quantum operators.

core/backend.py

PyTorch backend: device and dtype globals (DEVICE, DTYPE), set_device(...), set_dtype(...), truncating SVD helper, Krylov matrix-exponential.

core/graph.py

Tree topology primitives: Point, Node, End, Frame. Tracks node adjacency, axis duality, and provides BFS/DFS visitors for traversals.

core/model.py

The Model class stores one PyTorch tensor per node and tracks the current gauge centre.

core/sop.py

SumOfProducts — canonical form Σ_k c_k ⊗_s O_s^(k). Each term is a product of local matrices keyed by DOF index. Supports batched contraction via opt_multitransform.

core/gauge.py

QR-based gauge moves that shift the gauge centre between neighbouring nodes during PS1 sweeps.

Layer 2 — physics/

The Lindblad-equation layer. Translates Hamiltonians and jump operators into a SoP Liouvillian, and wires up TTN topologies appropriate for an open quantum system.

physics/liouvillian.py

LiouvillianSoP — builds the Liouvillian from a Hamiltonian and a list of jump operators. pauli_decompose(...) expands every operator in the Pauli basis and lifts the result to local 4×4 superoperators.

physics/liouville.py

Liouville-space utilities: vec(ρ) / unvec(v), trace_from_vec(...), expect_from_vec(...). Used for I/O at the boundaries of the TTN propagation.

physics/frame_factory.py

LindbladFrameFactory.train() and .balanced_tree() build the appropriate TTN topology and the DOF-to-End mapping for the given number of qubits.

Layer 3 — propagation/

The TDVP engine. Drives the TTN forward in time, given a SoP Liouvillian.

propagation/tdvp.py

TDVPPropagator — the workhorse class. Implements both strategies: PS1 (one-site projector-splitting sweeps with QR gauge transformations) and VMF (whole-network ODE integration via torchdiffeq).

propagation/effective_h.py

SimpleEffectiveH — bottom-up contraction of all child sub-trees into a local effective Liouvillian acting on a single node. Used at every step of a PS1 sweep.

propagation/integrators.py

krylov_expm_apply(...) — Krylov-subspace approximation of $\exp(-i\,\mathcal{L}_{\text{eff}}\,t)\,|\psi\rangle$ for the on-node and on-bond evolutions inside PS1.

User-facing entry point

Everything above is wrapped in one class:

solver.py

LindbladTTN is the only public class. Its constructor accepts $H_0$, drives, jump operators, and TTN/TDVP options; .run() propagates and returns a LindbladResult. See the API reference for every argument.

The companion time_dependent.py module composes static and time-dependent SoP terms into the time-evaluated Liouvillian at each step — supports both the single-drive shortcut (f, V) and arbitrary drives=[(f_i, V_i), …] lists.

Data flow at a glance

  1. Build — User passes Hamiltonian, jumps, topology, bond_dim to LindbladTTN(...). physics turns operators into a SoP Liouvillian; physics.frame_factory builds a TTN topology; core represents it.
  2. Initial state — solver.run(rho0=…) compresses the vectorised $|\rho_0\rangle\!\rangle$ into the TTN via sequential SVD (an MPS-style sweep).
  3. Propagate — At every step, propagation.tdvp applies the chosen TDVP strategy. For time-dependent problems, time_dependent rebuilds the SoP at the step midpoint.
  4. Extract — Periodically (every save_every steps), the TTN is contracted into a flat vector, observables are computed in Liouville space, and the results are appended to the LindbladResult.
