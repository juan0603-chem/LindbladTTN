Why LindbladTTN

A scalable open-system solver

LindbladTTN exposes a single class. Inside, it builds the Liouvillian as a Sum-of-Products operator, decomposes the density matrix as a Tree Tensor Network, and evolves it with a TDVP integrator — all on top of PyTorch.

TTN Compression

The density matrix lives in a tree of low-rank tensors. Bond dimension controls the accuracy/cost tradeoff — pick the rank that fits your problem.

PS1 & VMF

Two complementary TDVP strategies: projector-splitting (PS1) for fixed-rank stability, and Variable Mean-Field (VMF) for adaptive ODE integration via torchdiffeq.

Tree & Train

Choose topology="train" for an MPS-style chain or "tree" for a balanced binary tree. The frame factory wires up either topology automatically.

Time-Dependent Drives

Pass f(t) and V for a single drive, or a list of (f_i, V_i) tuples for multi-pulse circuits — every channel becomes a SoP term.

GPU via PyTorch

The same code runs on CPU or CUDA — set device="cuda". On GPU, drop to dtype="complex64" for a ~3× speedup at ~10⁻⁵ accuracy cost.

QuTiP-Validated

Every example ships with a side-by-side QuTiP comparison. Single-qubit, driven, and multi-qubit dynamics all agree with qutip.mesolve to ~10⁻⁷.

Quick Start

A simulation in four steps

Define the system, build the solver, propagate, inspect.

import numpy as np
from lindblad_ttn import LindbladTTN

# Pauli operators
sz = np.array([[1,  0], [0, -1]], dtype=complex)
sm = np.array([[0,  0], [1,  0]], dtype=complex)  # sigma minus

# Single-qubit decay: H = omega/2 sz, jump = sqrt(gamma) sigma-
omega, gamma = 1.0, 0.3
H0    = 0.5 * omega * sz
L_ops = [(gamma, sm)]
solver = LindbladTTN(
    H0=H0,
    f=None, V=None,         # no drive in this example
    L_ops=L_ops,
    n_sites=1,
    bond_dim=4,
    topology="train",
    strategy="ps1",
    device="cpu",
    dtype="complex128",
)
# Initial state: excited |1><1|
rho0 = np.array([[1.0, 0.0], [0.0, 0.0]], dtype=complex)

result = solver.run(
    rho0=rho0,
    t_span=(0.0, 15.0),
    dt=0.01,
    observables=[sz],
    save_every=1,
    verbose=True,
)
import matplotlib.pyplot as plt

print("Final trace:", result.norm[-1])
print("Final density matrix:\n", result.rho_final)

plt.plot(result.times, result.expect[0].real, label=r"$\langle\sigma_z\rangle$")
plt.xlabel("t"); plt.ylabel("expectation")
plt.legend(); plt.show()

# Save full report to plain text
result.save_txt("decay.txt")

Architecture

Three layers, one class

Everything users touch is LindbladTTN. Underneath, three independent layers do the work.

core/

TTN infrastructure: backend, graph topology (Node/End/Frame), per-node tensor storage, gauge transformations, Sum-of-Products operators.

physics/

The Lindblad layer: Pauli decomposition of the Liouvillian into local 4×4 superoperators, Liouville-space vec/unvec, train and balanced-tree frame factories.

propagation/

TDVP engine: PS1 projector-splitting sweeps, VMF whole-tensor ODE integration, Krylov matrix exponentials, local effective Liouvillians.
