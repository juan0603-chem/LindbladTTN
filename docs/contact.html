Issues & bug reports

The fastest path is the GitHub issue tracker. Please include:

  • A minimal reproducing example (smallest n_sites / bond_dim / time span that triggers the issue).
  • The full traceback (or a description of the wrong behaviour).
  • Your Python, PyTorch, and CUDA versions.
  • Whether you're on CPU or GPU.

Open an issue

Contributing

LindbladTTN follows a few simple conventions:

  • Tests first. New physics or algorithms should add a QuTiP cross-check to tests/.
  • Docstrings in NumPy style. Public functions/classes get a one-line summary, a Parameters block, and a Returns block.
  • One feature per PR. Keeps reviews fast and bisects easy.

For a discussion before opening a PR (especially for larger changes), file an issue first.
